# Coscos i18n/L10n Implementation Plan

## Overview
- **Library**: next-intl (Next.js 최적화된 i18n 라이브러리)
- **Supported Languages**: Korean (ko), English (en)
- **Default Language**: Korean (ko)
- **Browser Detection**: Yes (브라우저 설정 기반 자동 감지)

## Architecture

```
web/
├── i18n/
│   ├── request.ts          # Server-side locale configuration
│   └── routing.ts          # Routing configuration
├── messages/
│   ├── ko.json             # Korean translations
│   └── en.json             # English translations
├── middleware.ts           # Locale detection middleware
└── app/
    └── [locale]/           # Locale-based routing
```

## Implementation Steps

1. **Install next-intl**
2. **Create translation files** (messages/ko.json, messages/en.json)
3. **Configure i18n routing and middleware**
4. **Create LanguageSelector component**
5. **Update layout.tsx with i18n provider**
6. **Apply translations to all pages/components**
7. **Test and verify browser language detection**
